# Key Concepts Learned

## Spec-Driven Development (SDD)

Traditional development often treats code as the primary artifact.

Spec-Driven Development reverses this approach:

```text
Specification
      ↓
Implementation
      ↓
Validation
      ↓
Deployment
```

The specification becomes the source of truth while generated code can evolve or be regenerated when necessary.

---

## Behavior-Driven Development

Instead of focusing on implementation details, teams define expected behavior.

Example:

```gherkin
Feature: Expense Approval

Scenario: Auto Approval

Given an expense below $100
When the expense is submitted
Then it should be automatically approved
```

This approach makes systems easier to validate and maintain.

---

## Code as a Disposable Artifact

One of the most important concepts introduced in this unit was:

> Code is temporary, specifications are permanent.

With modern AI-assisted development workflows, code can be regenerated from well-defined specifications whenever needed.

---

## Zero-Trust AI Development

Production AI systems should never assume trust.

Every action should be verified through:

* Policy Validation
* Security Checks
* Authentication
* Evaluation
* Human Oversight

---

## Automated Code Review Agents

AI agents can participate in development pipelines by:

* Reviewing Pull Requests
* Checking Security Policies
* Detecting Vulnerabilities
* Enforcing Standards

This enables scalable software development practices.

---

## Policy Servers

Policy servers provide centralized control over AI agent behavior.

Responsibilities include:

* Authorization
* Compliance Enforcement
* Risk Mitigation
* Security Validation

A managed environment that executes AI agents at scale.

Benefits:

## Cloud Run
